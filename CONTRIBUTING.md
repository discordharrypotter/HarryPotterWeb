# Contributing

해리포터 봇 웹사이트에 기여해 주셔서 감사합니다.
가이드 문서 개선, 오탈자 수정, 새로운 기능 추가 등 다양한 방식으로 기여할 수 있습니다.

## 기여 방법

1. [GitHub 저장소](https://github.com/discordharrypotter/HarryPotterWeb)를 포크합니다.
2. 포크한 저장소에서 새 브랜치를 생성합니다.
3. 변경 사항을 커밋하기 전에 `yarn lint`를 실행하여 포맷과 타입 오류가 없는지 확인합니다.
4. PR 제목은 변경 내용을 간결하게 요약합니다.
5. CI가 통과해야 병합 가능합니다.

### 기여 시 유의사항

- 기존 문서의 스타일과 형식을 따라 일관성을 유지해 주세요.
- 명확하고 간결하게 작성하여 모든 이용자가 쉽게 이해할 수 있도록 해주세요.
- 기존 문서와 중복되는 내용이 없는지 확인해 주세요. 중복이 있다면 기존 문서를 개선하는 방향으로 기여해 주세요.
- 리뷰와 병합 과정이 필요하므로 반영까지 시간이 걸릴 수 있습니다.

## 개발 환경

- **Node.js** 22+
- **Yarn** (패키지 매니저)
- **SvelteKit** + **Cloudflare Pages**

```bash
# 의존성 설치
yarn install

# 개발 서버 실행
yarn dev

# 타입 체크
yarn check

# 포맷 검사
yarn format:check

# 포맷 자동 수정
yarn format

# 프로덕션 빌드
yarn build
```

## 가이드 문서 추가

`content/guide/` 디렉토리에 마크다운 파일을 추가하면 자동으로 가이드에 반영됩니다.

### 파일 및 폴더 네이밍 규칙

- 폴더명과 파일명은 **소문자 영어**로 작성합니다.
- 두 단어 이상인 경우 하이픈(`-`)으로 구분합니다. (예: `getting-started`)
- 이미지 파일은 `static/images` 폴더에 `{사용 위치}-{순서}` 형식으로 저장합니다. (예: `getting-started-invite-0.png`)

### 폴더 (카테고리) 추가

`content/guide/{폴더명}/_meta.json` 파일을 생성합니다.

```json
{
  "title": "표시 이름",
  "icon": "아이콘",
  "order": 0
}
```

### 페이지 추가

`content/guide/{폴더명}/{파일명}.md` 파일을 생성합니다.

```markdown
---
title: "제목"
description: "설명"
order: 0
---

본문 내용
```

### 마크다운 확장 문법

| 문법 | 설명 |
|---|---|
| `**텍스트**` | 굵게 |
| `__텍스트__` | 밑줄 |
| `{#HEX 텍스트}` | 컬러 칩 (색상 점 + 컬러 텍스트) |
| `[>텍스트](url)` | 버튼 링크 |
| `![이름](icons/파일.png)` | 인라인 아이콘 (20x20) |
| `>>> 제목` ~ `<<<` | 드롭다운 (접기/펼치기) |

## 코드 스타일

- Prettier로 포맷을 관리합니다. `.prettierrc` 설정을 따릅니다.
- 탭 들여쓰기, 작은따옴표, trailing comma를 사용합니다.
